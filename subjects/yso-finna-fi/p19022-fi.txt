MySQL-tietokannan siirto MS SQL Server -tietokantaan
Tämän opinnäytetyön tavoitteena on siirtää olemassa oleva MySQL-tietokanta Microsoft SQL Server-tietokantaan. Siirtoprosessissa käytetään hyväksi Microsoft SQL Server Migration Assistant-ohjelmaa soveltuvin osin. Siirrettävä tietokanta toimii selainpohjaisen tuntikirjausten seurantajärjestelmän pohjana. Järjestelmä ei ole vielä tuotantokäytössä, eikä käytettävissä olevista tietokanta-alustoista ole ollut tietoa järjestelmän kehitysvaiheessa. Tästä syystä on valmistauduttava tilanteeseen, jossa tämänhetkistä tietokanta-alustaa ei voida syystä tai toisesta käyttää tuotantoympäristössä, vaan järjestelmä on siirrettävä toiselle tietokanta-alustalle. Opinnäytetyö on rajattu koskemaan pelkästään tietokannan siirtoa tietokanta-alustalta toiselle. Lähdejärjestelmä kuvataan yleisellä tasolla, mutta järjestelmän teknistä toteutusta ei tämän opinnäytetyön puitteissa käydä läpi. Opinnäytetyön viitekehyksessä käsitellään relaatiotietokannan teoreettisena pohjana toimivaa relaatiomallia, SQL-kieltä sekä empiirisessä osassa käytettäviä tietokantaohjelmistoja. Empiirisessä osassa suunnitellaan ja toteutetaan siirtoprosessi, jolla lähdetietokanta voidaan siirtää kohdetietokantaan niin, että kohdetietokanta on rakenteeltaan ja toiminnallisuudeltaan lähdetietokantaa vastaava. Siirtoprosessin suunnitteluvaiheessa tullaan tulokseen, että siirtoprosessia ei voida suorittaa kokonaisuudessaan Microsoft SQL Server Migration Assistant-ohjelman avulla. Vaihtoehtoiseksi toteutustavaksi valitaan T-SQL-kielisen skriptin kirjoittaminen jonka avulla siirtoprosessi voidaan suorittaa kokonaisuudessaan automaattisesti. Opinnäytetyön tuloksena saadaan toteutettua työn tavoitteena oleva siirtoprosessi onnistuneesti. Tietokanta-alustan vaihdolla saavutetaan pientä suorituskyvyn parannusta lisättäessä tietokantaan uutta tietoa. Siirtoprosessin mielekkyys asettuu työn aikana uuteen valoon. Mikäli siirtoprosessille ei ole pakottavaa tarvetta, kannattaa se suorittaa vain siinä tapauksessa, että MS SQL Serveristä on käytettävissä vähintään Standard-versio.

Tämän opinnäytetyön tavoitteena oli testata sekä valmistella organisaation Lync-tuotantoympäristö organisaation käyttöä varten. Opinnäytetyön toimeksiantaja on Pohjois-Pohjanmaan sairaanhoitopiiri. Tarve työlle lähti liikkeelle Pohjois-Pohjanmaan sairaanhoitopiirin tarpeista kehittää verkkoviestintäratkaisujaan. Organisaation vaatimuksena oli kuva- ja äänituella varustettu verkkokokousväline, joka soveltuisi myös dokumenttien esittämiseen ja jakamiseen kokouksissa. Verkkokokousvälineen tulisi olla käytettävissä sijainnista riippumatta, myös tuki ulkopuolisille kokouksiin osallistujille oli tarpeen. Lisäksi pikaviestitoiminnot sekä tavoitettavuustiedon jakaminen koettiin tärkeäksi. Lync-ympäristön testaaminen sekä sen kehittäminen perustui suurilta osin oman kokemuksen kautta syntyneisiin havaintoihin. Työn tukena oli vahvasti myös opinnäytetyön lähteinä käytetty kirjallisuus, mutta suuri osa testauksen tuomista tuloksista perustui kuitenkin itse tehtyihin havaintoihin. Opinnäytetyön tuloksena saavutettiin Microsoft Lync Server 2013 -ydinominaisuuksilla varustettu toimiva Lync-tuotantoympäristö. Ympäristön toimivuuden takaamiseksi vaadittiin oikein konfiguroidut palvelimet sekä Lync-asiakasohjelmisto. Lync-ympäristön toimivuuden määrittäminen perustuu vahvasti Lync-palvelinohjelmiston hallintaan. Automaatioita vaativien toiminallisuuksien johdosta, opinnäytetyön tuloksena syntyi myös PowerShell-skriptejä. Skriptien avulla ympäristön Lync-käyttöoikeuksien hallinta saatiin automatisoitua. Lisäksi opinnäytetyön tuloksena syntyi Lync-asiakassovellusta testanneille käyttäjille suunnattu käyttökokemuskysely, sekä siitä syntyneiden vastausten analysointi. Käyttäjille tehtiin myös Lync 2013-asiakassovelluksen käyttöönottoa sekä käytön aloitusta avustava Lync 2013 -käyttöohje. Jatkoa ajatellen, opinnäytetyö jätti mahdollisuuden ympäristön jatkokehitykselle. Vikasietoisuuden sekä korkean käytettävyyden takaamiseksi ympäristöä on mahdollista laajentaa uusilla palvelimilla. Lisäksi Lync Server 2013 tarjoaa lisäominaisuuksia, mitä ei otettu vielä käyttöön tämän opinnäytetyön aikana. Osaksi Lync-toiminnallisuutta on mahdollista ottaa käyttöön pysyvät keskusteluhuoneet tai lähteä tukemaan Lync-mobiilisovellusten käyttöä. Lisäksi Lync on mahdollista integroida osaksi perinteistä puhelinverkkoa (PSTN), joka taas avaa uusia ulottuvuuksia käytön suhteen.

Tämän opinnäytetyön tavoitteena oli ottaa käyttöön uusi Microsoft SharePoint 2010 Server -pohjainen intranet-ratkaisu Helsinkiläisessä oppilaitoksessa. Oppilaitoksella on paljon asiakirjoja ja tiedostoja eri paikoissa, niin intranet-sivustolla, kuin ulkoisella verkkoasemallakin ja sen tulevaisuuden tavoitteena on siirtää nämä kaikki SharePoint -palvelun alle.    Oppilaitos käyttää Microsoft Office 2010 -ohjelmistoja, ja nämä sulautuvat saumattomasti yhteen SharePoint 2010:n kanssa. Esimerkiksi Microsoft Word 2010:lla voidaan tallentaa suoraan SharePoint 2010 -kirjastoon.    Tämän opinnäytetyön tuloksena oppilaitokselle valmistui asennettu ja asetuksiltaan toimivaksi määritetty SharePoint 2010 -palvelin. Lisäksi oppilaitokselle luotiin uusi SharePoint-pohjainen intranet-sivusto, joka muokattiin rakenteeltaan oppilaitoksen henkilökunnan ohjeiden mukaiseksi. Sivustolle luotiin haluttua toiminnallisuutta, esimerkiksi yhteisen Outlook-kalenterin muodossa.    Projekti oli monivaiheinen. Aluksi luotiin virtuaaliset palvelimet, joihin asennettiin tarvittavat palvelinohjelmistot. Tämän jälkeen asennettiin varsinainen SharePoint 2010 Server -ohjelmisto. Kun asennus oli valmis, SharePoint 2010:n asetukset käytiin läpi ja oppilaitokselle luotiin sivustokokoelma. Tästä sivustokokoelmasta muokattiin uusi intranet-sivusto. Web-osia käyttäen sivustolle luotiin haluttua sisältöä.    Lisäksi tämän opinnäytetyön aikana oppilaitoksen henkilökunnan kanssa pidettiin palavereja, joissa määriteltiin sivustolle tuotavien asiakirjojen vaatimia metatietoja.
