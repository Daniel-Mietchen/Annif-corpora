Tässä Opinnäytetyössä käsitellään Windows-käyttöjärjestelmien asetusten keskitettyä hallintaa ryhmäkäytäntöteknologian avulla. Työssä lähdettiin selvittämään miten ryhmäkäytäntöteknologiaa voisi hyödyntää entistä tehokkaammin Sisu Akselit Oy:n lähiverkossa. Lopputuloksena luotiin yritykselle uusi ryhmäkäytäntötoteutus, jossa tutkimustulokset sovellettiin käytäntöön. Työn tavoitteena oli saada yhä suurempi määrä käyttöjärjestelmien asetuksista keskitetyn hallinnan piiriin, jolloin järjestelmänvalvojan ei enää tarvisi kiertää yksitellen jokaista tietokonetta asetuksia konfiguroidessaan.  Ryhmäkäytäntöteknologian ansioista asetusten asettaminen saatiin automatisoitua; ne konfiguroituivat tietokoneille käynnistyksen yhteydessä ja käyttäjien sisäänkirjautuessa työasemille. Lisäksi teknologia pakotti asetukset tietokoneille, jolloin käyttäjät eivät kyenneet muuttamaan keskitetysti asetettuja asetuksia. Näin ympäristöstä saatiin entistä vakioidumpi. Työ jakautui tutkimus- ja toteutusvaiheisiin. Tutkimusvaiheessa perehdyttiin teknologian toimintaan sekä sen kokemiin parannuksiin, ja selvitettiin onko parannuksia mahdollista hyödyntää yrityksen nykyisessä ympäristössä. Toteutusvaiheessa luotiin yritykselle uusi ryhmäkäytäntötoteutus, jossa teknologiaa hyödynnettiin lähtötilannetta tehokkaammin. Sekä tutkimus- että toteutusvaiheissa tukeuduttiin teknologian toimintaan perehtyneiden asiantuntijoiden julkaisemaan kirjallisuuteen sekä verkkoartikkeleihin. Työn tuloksena luotiin yritykselle uusi ryhmäkäytäntötoteutus, jossa Windows-käyttöjärjestelmien asetuksista huomattavasti suurempaa osaa hallittiin keskitetysti ryhmäkäytäntöteknologian avulla. Uudessa toteutuksessa otettiin käyttöön suuri määrä aikaisemmin hyödyntämättömiä ryhmäkäytäntöasetuksia sekä teknologisia parannuksia, koska tutkimusvaiheessa oli selvinnyt, että myös näitä voitiin hyödyntää yrityksen olemassa olevassa ympäristössä. Näin ryhmäkäytäntöteknologian vaikutuspiiriin saatiin entistä suurempi määrä Windows-käyttöjärjestelmien asetuksista. Uudella ryhmäkäytäntötoteutuksella varauduttiin myös Windows 7 -käyttöjärjestelmän käyttöönottoon yrityksen työasemissa lähitulevaisuudessa. Saatuja tuloksia ei voida yleistää yritykselle luodun uuden ryhmäkäytäntötoteutuksen osalta. Kyseessä oli yksittäinen yritys, ja sen toimintaa parhaiten tukevan ryhmäkäytäntötoteutuksen suunnittelu ja luonti. Tutkimusvaiheessa selvitettyjen teknologisten parannusten, sekä uuden toteutuksen suunnittelussa hyödynnettyjen asiantuntijoiden suositusten osalta tulokset ovat kuitenkin yleistettävissä ja siirrettävissä myös muihin teknologiaa hyödyntäviin ympäristöihin.

Tämän opinnäytetyön tarkoituksena on tutustua Microsoftin käyttöjärjestelmään Windows 7:ään ja tutkia sen soveltuvuutta toimeksiantajalle. Työn tarkoituksena oli tutkia mitä muutoksia ja parannuksia käyttöjärjestelmässä on tapahtunut edellisen käyttöjärjestelmäversion Windows Vistan jälkeen. Testikäytössä oleva Windows 7 on betaversio eli kokeiluversio. Opinnäytetyö tehtiin O.K. Auto Oy:lle.     Käyttöjärjestelmiä on useita, mutta yleisin käyttöjärjestelmä on Microsoft Windows. Microsoft julkaisee vuoden 2009 lopussa tai vuoden 2010 alkupuolella uuden käyttöjärjestelmän Windows 7:n. Windows 7:ltä odotetaan paljon, koska Windows Vista oli pettymys. Pystyykö uusi Windows käyttöjärjestelmä olemaan Vistaa nopeampi ja varmatoimisempi?       Opinnäytetyössä käydään läpi Windows 7:n asentaminen ja toimivuus.  Tutkimuksen tehtävänä on selvittää, kuinka O.K. Auto Oy:n henkilökunta pystyy toimimaan vanhan käyttöjärjestelmän kanssa ja millä tavalla uusi käyttöjärjestelmä vaikuttaisi työntekoon.    Tutkimuksessa tuli selville, että Windows 7 ei eroa edellisestä käyttöjärjestelmästä kovinkaan paljon. Pieniä parannuksia Windows 7:stä kuitenkin löytyy, kuten esimerkiksi työpöydän uusi alapalkki. Visuaalinen ilme on pysynyt ennallaan. Vaikka Windows 7 ei tarjoa paljoa uutta, on se kuitenkin edeltäjäänsä Windows Vistaa nopeampi ja vakaampi käyttää.     Windows 7 soveltuisi O.K. Auto Oy:lle vakautensa ja monipuolisuutensa vuoksi. Kyselytutkimuksessa ilmeni, että O.K. Auto Oy:n henkilökunnalla on hyvät atk taidot ja he oppisivat uuden käyttöjärjestelmän käytön muutamassa viikossa.     Asiasanat: Microsoft, Windows, Vista, Windows 7.

Projekti tehtiin eräälle Helsingin seudun oppilaitokselle. Projektin ideana oli päivittää oppilaitoksessa käytössä oleva vanhentunut Windows XP x86 käyttöjärjestelmä uuteen Windows 7 x64 käyttöjärjestelmään Configuration Manager 2007:n avulla. Configuration Manager 2007 oli jo aikaisemmin otettu käyttöön oppilaitoksessa. Projekti koostui suunnittelusta, toteutuksesta ja testauksesta. Suunnittelussa määritettiin mitä tietokonemalleja projekti koskee. Toteutusvaiheessa luotiin levykuva sekä haettiin tarvittavat laiteajurit. Testausvaiheessa käyttöjärjestelmän jakamista testattiin eri tyyppisiin testikoneisiin. Testauksen jälkeen järjestelmä otettiin käyttöön. Opinnäytetyön alussa käsitellään mitä muutoksia oppilaitoksessa on jo tehty ja mitä oppilaitoksessa pitäisi vielä tehdä. Lisäksi opinnäytetyön alussa käsitellään miten projekti tulee edistymään. Opinnäytetyön loppuvaiheessa käsitellään miten projekti eteni, miten eri konetyypeille löytyi ajureita sekä miten asennusvaiheet onnistuivat. Lisäksi työn tarkoituksena oli tuottaa oppilaitokselle ohjeistus Windows 7:n käyttöönotosta Configuration Manager 2007:n avulla. Nämä ohjeet löytyvät tämän projektin liitteistä. Projektin lopputuloksena syntyi toimiva tuote, jolla voidaan nopeasti siirtää Windows 7 x64 levykuva työasemille Configuration Manager 2007:n avulla.  Projektia voidaan kehittää vielä nostamalla Configuration Managerin käyttöastetta sekä lisäämällä eri konemallien laiteajureita. Lisäksi projektia voidaan kehittää tuomalla Configuration Managerin käyttöön uusia käyttöjärjestelmälevykuvia.

Windows XP on ollut yritysmaailmassa käytössä aivan näihin päiviin saakka. Yrityksissä käytetään useita eri ohjelmia ja ratkaisuja, joista osa ei välttämättä ole yhteensopivia uudemman käyttöjärjestelmän kanssa. Tämä on osaltaan hidastanut yrityksien siirtymistä uuteen käyttöjärjestelmään aikaisemmin. Microsoftin tuki XP-käyttöjärjestelmälle on päättymässä huhtikuussa 2014, minkä johdosta viimeisetkin yritykset ovat aloittaneet Windows 7 -käyttöönottoprojektit kiireellisillä aikatauluilla. Tämän opinnäytetyön aiheena oli määritellä, suunnitella, toteuttaa ja testata Windows 7 -yhteensopiva ohjelmiston jakelumenetelmä. Työn toimeksiantaja toimittaa palvelin- ja työasemaohjelmista koostuvaa ohjelmistokokonaisuutta asiakkailleen. Asiakasorganisaatioiden ATK-ylläpito asentaa jakelumenetelmän avulla työasemaohjelmistoon kohdistuvia päivityksiä keskitetysti ja etänä. Perinteinen ohjelmiston jakelumenetelmä ei ollut yhteensopiva Windows 7 -käyttöjärjestelmän kanssa, mihin tässä työssä lähdettiin hakemaan ratkaisua. Ratkaisu päätettiin toteuttaa Windows PowerShellin avulla, joka on tullut osaksi käyttöjärjestelmää Windows 7:stä lähtien. Kehitystyö tehtiin PowerShellin omalla PowerShell ISE -työkalulla. Ohjemateriaalina käytettiin Microsoftin MSDNsivuston artikkeleita, internetin hakukoneita sekä PowerShellin omaa ohjetietokantaa. Työn lopputuloksena syntyi Windows 7 -yhteensopiva uusi ohjelmiston jakelumenetelmä, jonka avulla ohjelmistopäivitykset voidaan asentaa ympäristön kaikille työasemille keskitetysti ja etänä. PowerShell osoittautui projektin aikana tehokkaaksi työkaluksi tietokoneiden ylläpidossa ja automatisoinnissa. PowerShell löytyy jatkossa kaikista Microsoftin uusista käyttöjärjestelmistä, joten se tulee ottamaan merkittävän roolin järjestelmien ylläpitotyössä.

Järjestelmä, joka päätettiin ottaa käyttöön Aalto Yliopiston kahdella laitoksella BECS:ssä (lääketieteellisen tekniikan ja Laskennallisen tieteen laitos) ja TFY:ssä (teknillisen fysiikan laitos), on nimeltään WPKG. WPKG on avoimen lähdekoodin asennustenhallinta järjestelmä. Opinnäytetyössä käydään läpi ohjelmien unattended asennukseen liittyviä tekniikoita ja erilaisia tapoja tehdä ohjelma-asennukset.    Tässä opinnäytetyössä taustoitetaan, että miksi BECS:ssä ja TFY:ssä päätettiin toteuttaa ohjelmistojen jakelu WPKG:n avulla, sekä kuvataan, että miten testattiin ja otettiin käyttöön ohjelmistojen keskitetty asennustenhallinta ja päivitys Windows 7-ympäristössä. Ennen tätä käytössä ei ollut vastaavaa järjestelmää.    Opinnäytetyön tavoitteena oli saada toimiva ohjelmistojen keskitetty asennustenhallinta ja päivitys 64-bittiseen Windows 7-ympäristöön ja tässä onnistuttiin hyvin.    WPKG sopii laitoksien ympäristöihin erinomaisesti. Se on helppokäyttöinen ja on vapauttanut työaikaa muuhun käyttöön huomattavasti.

Opinnäytetyön tarkoituksena oli suunnitella Rapico Ay:n tietojärjestelmän päivitys Windows XP:stä Windows 7:ään. Suunnitelma käsittää Rapicon uudet tietokoneet käyttöjärjestelmineen, ohjelmistot sekä tarvittavat lisälaiteet. Rapico Ay on vuonna 2004 perustettu pienkoneiden huoltoon ja korjaukseen erikoistunut yritys, jonka asiakkaisiin kuuluvat rakennusliikkeet, yksityiset sekä yritykset. Rapicolla huolletaan paineilma, akku- ja sähkökoneita sekä polttomoottorikoneita. Monimerkkihuollon lisäksi he tarjoavat myös takuuhuoltoja tunnetuille pienkonemerkeille. Opinnäytetyössä perehdytään tietokoneen rakenteeseen sekä selitetään tärkeimpien yksittäisten osien omaa toimintaa sekä niiden toimintaa suhteessa muihin tietokoneen osiin. Ohjelmia sekä ohjelmistoja käsitellään yleisellä tasolla. Käyttöjärjestelmien toimintaa on myös selitetty ja Windows- sekä Linux-käyttöjärjestelmien historiaa ja niiden eri versioiden erovaisuuksia käydään läpi. Virustorjuntaohjelmistoilla on myös oma osansa tässä opinnäytetyössä. Opinnäytetyössä on pyritty esittämään mahdollinen malli, minkä perusteella tulevia tietokone-, lisälaite- ja ohjelmistohankintoja voisi suunnitella tarkemmin. Tavoitteena oli luoda eräänlainen ohjenuora siihen, mitä Rapicon tulevissa Windows 7-tietojärjestelmään liittyvissä hankinnoissa kannattaa ottaa huomioon.
