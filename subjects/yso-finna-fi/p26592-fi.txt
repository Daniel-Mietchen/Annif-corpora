Projektinhallintaohjelmiston hankinta sovelluksen käyttöönottoprojekteihin
Tavoitteenaan oli kehittää ja selkeyttää tilajan käytössä olevaa projektinhallintapohjaa, sekä löytää mahdollisesti uusia ratkaisuja työn toteuttajan avulla. Tavoitteena oli myös etsiä, vertailla ja evaluoida projektien hallintaan käytettäviä ohjelmistoratkaisuja, joita myöhemmin voitaisiin ottaa  käyttöön. Tilaajan projektinhallinnan organisoinnin tehostamiseksi. Uuden projektinhallintapohjan teoreettisena pohjana käytettiin, kappaleessa kuusi esiteltävää kieltä. Toimeksiantajan pyynnöstä kielen standardeista ei pidetty täysin kiinni, vaan niitä käytettiin soveltaen vertailukohtana toimeksiantajan uuden projenktinhallintapohjan luonnissa. Uuden pohjan tuli pääpiirteiltään mukailla vanhan pohjan toteuttamisjärjestystä, joka oli jaettu kolmeen päävaiheeseen. Pyrkien uudessa pohjassa selkeämpään, yleisluontoisempaan ja tilaajan tarpeisiin sopivan ratkaisun löytämiseen. Ohjelmistoratkaisujen valinnan kriteereinä oli ohjelmien vapaa saatavuus, sekä internet-pohjainen käyttöliittymävalmius etätyöskentelyn mahdollistamiseksi. Käytännössä tämä tarkoitti avointa lähdekoodia hyödyntävien ilmaisten ratkaisujen löytämistä, tilaajan ohjelmistolisenssikulujen karsimiseksi. Etäkäyttöominaisuuden johdosta ainoiksi vaatimuksiksi etäpäätelaitteelle jäisi yhteensopivan internetselaimen, sekä kiinteän tai langattoman internetyhteyden käyttömahdollisuus. Ohjelmistoratkaisu ei myöskään saanut olla liian raskas, tilaajan käyttötarpeisiin suhteutettuna. Kriteereiden täyttymistä valvottiin säännöllisten lähi- ja etäpalaverien avulla. Työn päätulokset olivat uusi projektinhallintapohja ja kaksi valittua ehdokasohjelmaa projektinhallinnan ohjelmistotyökaluksi. Toteuttaja suosittelee kuitenkin kahden ehdokasohjelman välistä tarkempaa jatkoevaluointia, ennen lopullisen valinnan tekemistä ohjelmistotyökalujen välilllä.

Tiivistelmä: Tässä opinnäytetyössä käsitellään Microsof System Center Configuration Manager 2012 –ohjelmistoa. Opinnäytetyön alussa tehdään yleiskatsaus Configuration Manager –ohjelmistoon ja sen suomiin mahdollisuuksiin. Käytännönosuudessa tutustutaan ohjelmiston asennukseen, käyttöjärjestelmän levitykseen ja ohjelmistokirjaston rakentamiseen.  Työ toteutettiin VMWare-virtualisointialustalla, minne luotiin kaksi palvelinta ja kolme työasemaa. Palvelimien käyttöjärjestelmiksi asennettiin Windows Server 2008 R2. Palvelimet jaettiin Domain Controlleriin ja Configuration Manageriin.  Configuration Manager palvelimelle asennettiin SQL-tietokanta, jonne koko ympäristö ohjattiin. Configuration Managerilla luotiin käyttöjärjestelmälevykuvat Windows 7 Enterprisesta, jotka asennettin kahdelle työasemalle verkon yli. Yksi työasema oli asennettuna valmiiksi, joka toimi ympäristön testauspisteenä.  Tämän opinnäytetyön tarkoituksena on saada yleiskäsitys Configuration Managerista ja sen monipuolisuudesta ympäristöstä riippuen.
